# Contributing

Thanks for considering contributing to NotebookLM Study Flow.

## Quick Start

1. Fork the repo and create a feature branch.
2. Make your changes.
3. Verify locally in Chrome (see README for local load steps).
4. Submit a pull request with a clear description.

## Development Notes

- Main logic lives in `src/content.js`.
- UI styles are in `src/styles.css`.
- Extension entry points: `manifest.json`, `src/background.js`.
- Test by reloading the extension in `chrome://extensions` and refreshing NotebookLM.

## PR Checklist

- [ ] Clear title and description of changes
- [ ] No unrelated formatting or refactors
- [ ] UI changes tested in NotebookLM
- [ ] README/CHANGELOG updated if needed

## Reporting Bugs

Please include:
- Steps to reproduce
- Expected vs. actual behavior
- Screenshots (if UI)
- Browser + OS versions
